# ─────────────────────────────────────────────────────────────────────────────
# Phansora API — consolidated dependencies (spokenverse + chrono_origin + dossier_nova)
#
# Torch baseline: aligned to the IndexTTS2 TTS engine (torch 2.8.0, CUDA 12.6).
# torchvision 0.23.0 / torchaudio 2.8.0 are the matching releases. Prod runs on an
# RTX A4000 (16 GB, Ampere) — plenty for IndexTTS2 (~5 GB). `+cu126` runs on any recent
# driver via CUDA minor-version compatibility. For a CPU-only host, swap `whl/cu126` +
# `+cu126` for `whl/cpu` + `+cpu` (but CPU synthesis is minutes/gen — GPU strongly preferred).
#
# System packages required: ffmpeg, tesseract-ocr
#
# TTS engine: IndexTTS2 is a git checkout (NOT a published pip package), run in-process.
# In prod: run `make install` (installs the torch/transformers pins below), then clone it
# and `pip install -e /var/www/index-tts` INTO this venv, install pynini via conda
# (`conda install -c conda-forge pynini==2.1.6`), download the IndexTeam/IndexTTS-2
# checkpoints, and set INDEXTTS2_REPO. The API boots without it; only TTS calls require
# it.  ⚠ IndexTTS2 pins `torch==2.8.*` / `transformers==4.52.1`, both satisfied by the
# pins below — so install IndexTTS2 AFTER make install and pip leaves torch untouched.
# ─────────────────────────────────────────────────────────────────────────────

# Torch wheels are pinned as DIRECT CloudFront URLs (download.pytorch.org) rather than
# via `--extra-index-url .../whl/cu126`. The index now hands out download-r2.pytorch.org
# (Cloudflare R2) links, and the prod network cannot complete a TLS handshake to
# Cloudflare (SSLV3_ALERT_HANDSHAKE_FAILURE / alert 40, IPv4 + IPv6). The
# download.pytorch.org path serves the same wheels from Amazon CloudFront, which is
# reachable. These URLs are pinned to cp310 / manylinux_2_28_x86_64 — the prod image
# (torch >= 2.7 uses the manylinux_2_28 tag, NOT the old linux_x86_64). For a CPU-only
# host, swap `whl/cu126` + `+cu126` for `whl/cpu` + `+cpu`.
torch @ https://download.pytorch.org/whl/cu126/torch-2.8.0%2Bcu126-cp310-cp310-manylinux_2_28_x86_64.whl
torchvision @ https://download.pytorch.org/whl/cu126/torchvision-0.23.0%2Bcu126-cp310-cp310-manylinux_2_28_x86_64.whl
torchaudio @ https://download.pytorch.org/whl/cu126/torchaudio-2.8.0%2Bcu126-cp310-cp310-manylinux_2_28_x86_64.whl

# TTS acceleration — DeepSpeed kernel-inject for the IndexTTS2 GPT/AR stage
# (INDEXTTS2_USE_DEEPSPEED=1). Pure-python wheel; its fused CUDA op JIT-compiles at
# model-load time, so it needs the CUDA *toolkit* (nvcc) present — NOT installable via
# pip. On the prod GPU box run `scripts/install-cuda-toolkit.sh` once, then set CUDA_HOME
# for the service (deploy/phansora-api.service.d/cuda-env.conf). CPU-only hosts can omit.
